Assumption Uncertainty

Monte Carlo over the assumptions register: every estimate becomes an honest range, every conclusion becomes a distribution — and the register turns into a prioritized to-do list of what to measure first.
SDA design document · 2026-07-03 · status: for owner review — not started
About this document & the project (context for the outside reader).

SDA (System Design Assistant) computes an architecture drawn on a canvas — load, latency, availability, cost — against declared requirements, and generates a design document whose assumptions register lists every input number with its provenance (documented / estimate / architect / default). This document adds the missing dimension: how much the conclusions wobble when the soft inputs move.

Monte Carloevaluating the same design thousands of times, each time with the uncertain inputs drawn from their declared ranges — the spread of results IS the uncertainty of the conclusion.
p95 / percentilethe value below which 95% of scenarios fall — "p95 cost $7.2k" means only 1 in 20 plausible worlds costs more.
tornado analysisranking the uncertain inputs by how much of the outcome's spread each one drives — the tall bars are what to measure first.
seededthe random draws come from a recorded seed, so every run is exactly reproducible.

1The last place the tool lies by tone

The generated document says the system costs $5,746.90 — five significant digits resting on a cache-hit ratio that is an estimate and service times that are estimates. The register honestly labels which inputs are soft; it stays silent about how much the conclusions inherit that softness. Point values with estimate-grade inputs are false precision — the polite name for a lie of tone.

2The model — declared ranges, deterministic sampling

PieceDesign
Range declarationsany config value may carry an optional range (data, like source/est.): {lo, hi} uniform or {lo, mode, hi} triangular — nothing fancier in v1 (heavier distributions need evidence we don't have; declaring them would itself be a guess). Catalog est. entries ship sensible sourced-or-est. ranges; the architect can add/override a range on ANY value from the Inspector — including their own declarations ("traffic 2,000, but could be 1,500–3,000").
SamplingN scenarios (default 1,000; knob up to 10,000): each draws every ranged input once, evaluates the full forward pass. Seeded → byte-reproducible; the seed is recorded in the document. No range anywhere ⇒ the feature is silent (no-filler) and results are today's, bit for bit.
Batch-evaluation seamthe ONE new abstraction: evaluateBatch(design, scenarios[]) → results[] — CPU worker first (a 6 ms evaluation × 1,000 ≈ 6 s off-thread), designed so a WebGPU backend slots in later. This seam IS layer 1 of the domain solver (TASK-79 "GPU proposes") — building this feature builds half the solver's foundation.
Outputsper metric (cost, flow latency, availability): median/p5/p95 + a histogram series; per SLO: % of scenarios satisfied; per ranged input: its tornado share of each outcome's variance (computed by correlation on the SAME sample — no extra runs needed in v1; one-at-a-time refinement later).
The 100% contractidentical to guarantees: certain (seeded sampling + the same deterministic engine per scenario), declared (every range is data with provenance — a register row), refused (no invented distributions; an input without a range simply stays fixed — the result then honestly reads "uncertainty of X not modeled").

3What the architect gains

PowerReads as
Cost/SLO as distributions"median $5.7k, p95 $7.2k" · "latency SLO holds in 97% of scenarios" — board-room language, computed
Tornado — what to measure first"80% of cost variance comes from the CDN hit ratio → measure it in production before committing; the rest isn't worth refining" — the register becomes an ACTION LIST
Risk-aware alternativesvariant A cheaper at median, B safer at p95 — the difference that most often decides real reviews, currently invisible everywhere
(Later) robust optimize"cheapest config holding the SLOs in ≥95% of scenarios" — designs stop being brittle to the day an estimate misses
